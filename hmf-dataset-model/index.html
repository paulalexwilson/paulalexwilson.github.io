Datasets provide a means for capturing and representing production, contextual and provenance information alongside the HMF tree metadata itself in such a way that it can be independently processed by user interfaces, storage subsystems and event streams.

Datasets are flattened representations of HMF resource graphs, so it is up to the client to frame them into a nested structure of their choosing.

Datasets used the concept of named graphs to capture supplementary information about a metadata tree. Named graphs paritition data into well-defined and unambigous graphs which can be independently processed or merged into a consolidate graphs as needed by downstream systems either programmatically or using frames.

HMF resources (nodes, properties, types, namespaces and so on) are identified using an IRI and multiple graphs may supply information about the same resource using that IRI.
